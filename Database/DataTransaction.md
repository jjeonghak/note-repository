# 트랜잭션
트랜잭션은 애플리케이션에서 몇 개의 읽기와 쓰기를 하나의 논리적 단위로 묶는 방법  
트랜잭션은 전체가 성공하거나 실패  

## ACID
트랜잭션이 제공하는 안전성 보장은 흔히 원자성, 일관성, 격리성, 지속성을 의미  

<br>

### 원자성(Atomicity)
일반적으로 더 작은 부분으로 쪼갤 수 없는 것을 의미  
여러 쓰기 작업이 하나의 원자적인 트랜잭션으로 묶여있고 결함 때문에 일부가 완료될 수 없다면 전체가 어보트  
즉 오류가 발생했을대 트랜잭션을 어보트하고 해당 트랜잭션에서 기록한 모든 내용을 취소하는 능력  

<br>

### 일관성(Consistency)
일관성의 아이디어는 애플리케이션 불변식 개념에 의존  
데이터베이스에서 불변식을 위반하는 잘못된 데이터를 쓰지 못하도록 방지 불가능  
데이터베이스에서 방지 가능한 부분은 그저 외래키 제약 조건이나 유일성 제약 조건  
즉 일관성은 정확하게는 애플리케이션의 속성  

<br>

### 격리성(Isolation)
동시에 실행되는 트랜잭션은 서로 격리된다는 것을 의미  
고전적인 곳에서는 격리성을 직렬성이라는 용어로 공식화  
각 트랜잭션이 전체 데이터베이스에서 실행되는 유일한 트랜잭션인 것처럼 동작한다는 것을 의미  
하지만 직렬성 격리는 성능 손해를 동반하기 때문에 현실에서는 거의 사용되지 않음  
실제로는 직렬성보다 보장이 약한 스냅샷 격리를 구현  

<br>

### 지속성(Durability)
트랜잭션이 성공적으로 커밋됐다면 하드웨어 결함이 발생하거나 데이터베이스가 죽더라도 기록한 모든 데이터는 손실되지 않는다는 보장  
비휘발성 저장소에 기록됐다는 뜻으로 일반적으로 데이터 구조가 오염됐을 때 복구 가능하게 해주는 쓰기 전 로그나 비슷한 수단을 동반
