# 파티셔닝
데이터셋이 매우 크거나 질의 처리량이 높다면 복제만으로는 부족  
데이터를 파티션으로 쪼개는 샤딩 작업이 필요  
파티션을 나눌 때는 보통 각 데이터 단위가 하나의 파티션에 속함  

<br>

## 파티셔닝과 복제

<img width="550" height="250" alt="partitioning_and_replication" src="https://github.com/user-attachments/assets/477077a3-d77d-4c9a-b9b4-ba95088787c7" />

보통 복제와 파티셔닝을 함께 적용해 각 파티션의 복사본을 여러 노드에 저장  
각 레코드는 정확히 한 파티션에 속하더라도 이를 여러 노드에 저장해서 내결함성 보장  
각 노드는 어떤 파티션에게는 리더이면서 다른 파티션에게는 팔로워 가능  

<br>

## 키-값 파티셔닝
파티셔닝의 목적은 데이터와 질의 부하를 노드 사이에 고르게 분산시키는 것  
파티셔닝이 고르게 이뤄지지 않아 다른 파티션보다 데이터와 질의가 많은 파티션은 쏠렸다고(`skewed`) 표현  
쏠림 현상이 발생해서 불균형하게 부하가 높은 파티션을 핫스팟(`hot spot`)이라고 표현  
핫스팟 회피를 하는 가장 간단한 방법은 레코드를 할당할 노드를 무작위로 선택하는 것  

<br>

### 키 범위 기준 파티셔닝

<img width="550" height="200" alt="range_based_partitioning" src="https://github.com/user-attachments/assets/9f6d353f-860a-4303-ae4d-48b64bad36ae" />

백과사전과 유사하게 각 파티션에 연속된 범위의 키를 할당하는 방식  
키 범위 크기가 반드시 동일할 필요 없이 데이터 분포에 따라 유동적으로 설정  
키 범위 검색 질의 효율이 좋음  
키 범위 기준 파티셔닝은 특정한 접근 패턴이 핫스팟을 유발하는 단점 존재  

<br>

### 키 해시값 기준 파티셔닝

<img width="550" height="200" alt="hash_based_partiitoning" src="https://github.com/user-attachments/assets/f54ca45b-117c-4774-97b3-8102cfb4806f" />

쏠림과 핫스팟의 위험을 방지하기 위해 해시 함수를 사용  
파티셔닝용 해시 함수는 암호적으로 강력할 필요 없음  
키 범위 검색 질의 효율을 지원하지 않음  

<br>

## 파티셔닝과 보조 색인
이전의 파티셔닝 방식은 키-값 데이터 모델에 의존  
보조 색인은 레코드를 유일하게 식별하는 용도가 아닌 특정한 값이 발생한 항목을 검색하는 수단  
많은 키-값 저장소에서 보조 색인을 지원하지 않지만 엘라스틱서치 같은 검색 서버에게는 존재의 이유  

<br>

### 문서 기준 보조 색인 파티셔닝

<img width="550" height="250" alt="document_partitioned_index" src="https://github.com/user-attachments/assets/801c8917-a804-4ff2-aedc-35d095fc4f7e" />

각 파티션은 자신의 보조 색인을 유지하며 그 파티션에 속하는 문서만 담당  
다른 데이터베이스 파티션을 신경쓰지 않는 지역 색인(`local index`) 방식  
클라이언트는 모든 파티션에 질의를 보내서 얻은 결과를 모두 병합 필수, 해당 방식이 스캐터/개더(`scatter/gather`) 방식  

<br>

### 용어 기준 보조 색인 파티셔닝

<img width="550" height="250" alt="term_partitioned_index" src="https://github.com/user-attachments/assets/3d682ac7-7a88-4043-839f-d6319a8b768b" />

각 파티션이 자신만의 보조 색인을 갖게 하고 모든 파티션의 데이터를 담당하는 전역 색인(`global index`) 방식  
용어 자체로 파티셔닝한 경우 범위 스캔에 유용하고, 용어의 해시값을 사용하면 파티셔닝 부하가 고르게 분산  
문서 파티셔닝 색인에 비해 읽기가 효율적  
전역 보조 색인은 대개 비동기로 갱신되기 때문에 데이터 갱신이 실시간으로 되지 않음  

<br>
